## 优化原则

1. **按功能分类**：相关文件放在同一文件夹
2. **统一命名**：遵循命名规范，保持一致性
3. **文档集中**：所有文档统一放在 Docs 文件夹
4. **数据分离**：数据文件单独管理
5. **代码归档**：废弃代码放在 Archive 文件夹

## 文件组织规范

### 代码文件
- **Services/**: 业务逻辑服务类
- **Models/**: 数据模型类
- **Extensions/**: 扩展方法
- **Helpers/**: 辅助工具类
- **Constants/**: 常量定义

### 非代码文件
- **SQL/**: SQL 脚本，按功能分类
- **Data/**: 数据源文件
- **Docs/**: 项目文档
- **Archive/**: 已废弃的代码
