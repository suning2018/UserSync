# 人员数据同步程序使用说明

## 概述

人员数据同步程序是一个基于 .NET 8.0 的控制台应用程序，用于将 `vps_empinfo_mes` 表中的数据同步到 `Sys_User` 和 `base_people` 表中，确保数据一致性。

## 功能模块

### 核心同步步骤（按执行顺序）

#### 步骤1：同步新用户到 Sys_User
- **功能**：将 `vps_empinfo_mes` 中在制的员工（`isactive=1`），且在 `Sys_User` 中不存在的员工同步到 `Sys_User`
- **说明**：自动生成新用户记录，包括生成GUID作为ID，设置默认的组织、部门、角色等信息

#### 步骤2：生成 Sys_UserLogOn 记录
- **功能**：为 `Sys_User` 中缺少 `Sys_UserLogOn` 记录的用户生成登录信息
- **说明**：自动生成 `F_UserSecretkey`（16位随机密钥），`F_UserPassword` 需要后续通过应用程序加密后更新

#### 步骤3：同步 base_people 记录
- **功能**：从 `vps_empinfo_mes` 同步数据到 `base_people` 表
- **说明**：为不存在的员工创建 `base_people` 记录，包括工号、姓名、在职状态等信息

#### 步骤4：关联 base_people 与 Sys_User
- **功能**：根据 `Sys_User.F_Account = base_people.Code` 更新 `base_people.SysUser = Sys_User.ID`
- **说明**：建立 base_people 与 Sys_User 的关联关系

### 其他同步任务（可选）

#### 同步 Sys_User.F_EnabledMark
- **功能**：根据 `vps_empinfo_mes.isactive=0` 更新 `Sys_User.F_EnabledMark=0`
- **说明**：将离职员工的用户状态标记为禁用

#### 同步 base_people.JobStatus
- **功能**：根据 `vps_empinfo_mes.isactive` 更新 `base_people.JobStatus`
- **映射规则**：
  - `isactive=0`（离职）→ `JobStatus=1`（离职）
  - `isactive=1`（在制）→ `JobStatus=0`（在制）

#### 同步 base_people.ManufactureGroup（可选）
- **功能**：根据 `personnel.group_name` 更新 `base_people.ManufactureGroup`
- **说明**：同步员工的组别信息

## 配置说明

### appsettings.json

```json
{
  "ConnectionStrings": {
    "SQLServer": "Server=服务器地址;Database=数据库名;User Id=用户名;Password=密码;TrustServerCertificate=true;Encrypt=false;Connection Timeout=30;"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft": "Warning"
    }
  },
  "LogSettings": {
    "LogPath": "Logs",
    "LogFileName": "personnel-sync-{Date}.log",
    "RetainDays": 30,
    "EnableDatabaseLog": true
  },
  "SyncSettings": {
    "CheckIntervalSeconds": 3600,
    "EnableAutoSync": true,
    "SyncTasks": {
      "SyncNewUsersFromVps": true,
      "GenerateUserLogOn": true,
      "SyncBasePeopleFromVps": true,
      "SyncBasePeopleSysUser": true,
      "SyncSysUserEnabledMark": true,
      "SyncBasePeopleJobStatus": true,
      "SyncBasePeopleManufactureGroup": false
    }
  }
}
```

### 配置项说明

- **ConnectionStrings:SQLServer**：SQL Server 数据库连接字符串
- **LogSettings**：日志配置
  - `LogPath`：日志文件保存路径
  - `LogFileName`：日志文件名（支持 `{Date}` 占位符）
  - `RetainDays`：日志保留天数
  - `EnableDatabaseLog`：是否启用数据库日志
- **SyncSettings**：同步配置
  - `CheckIntervalSeconds`：检查间隔（秒），默认 3600 秒（1小时）
  - `EnableAutoSync`：是否启用自动同步（持续监控模式）
  - `SyncTasks`：各同步任务的启用状态

## 使用方法

### 方法1：直接运行（推荐）

1. **配置数据库连接**
   - 编辑 `appsettings.json`，设置正确的数据库连接字符串

2. **配置同步任务**
   - 在 `appsettings.json` 的 `SyncSettings:SyncTasks` 中启用/禁用需要的同步任务

3. **运行程序**
   ```bash
   dotnet run
   ```
   或
   ```bash
   dotnet build
   dotnet bin/Debug/net8.0/UserSync.dll
   ```

4. **程序运行模式**
   - 如果 `EnableAutoSync = true`：程序将持续运行，按配置的间隔自动执行同步
   - 如果 `EnableAutoSync = false`：程序执行一次同步后退出

### 方法2：作为 Windows 服务运行

1. **安装服务**
   - 使用 NSSM 或其他服务管理工具将程序安装为 Windows 服务

2. **配置自动启动**
   - 在服务管理器中设置服务为自动启动

### 方法3：使用任务计划程序

1. **创建任务**
   - 打开 Windows 任务计划程序
   - 创建基本任务

2. **配置触发器**
   - 设置触发器（例如：每天、每小时等）

3. **配置操作**
   - 操作：启动程序
   - 程序：`dotnet.exe`
   - 参数：`D:\路径\UserSync.dll`
   - 起始于：程序所在目录

## 日志说明

### 文件日志

- **位置**：`Logs/personnel-sync-{Date}.log`
- **格式**：`{Timestamp} [{Level}] {Message}{NewLine}{Exception}`
- **保留**：默认保留 30 天

### 数据库日志

- **表名**：`SystemLog`
- **字段**：LogTime, LogLevel, Category, Message, Exception, Source, FileName, Operation, UserId, AdditionalData
- **启用**：通过 `LogSettings:EnableDatabaseLog` 配置

## 故障排查

### 1. 数据库连接失败

**问题**：程序启动时提示数据库连接失败

**解决方案**：
- 检查 `appsettings.json` 中的连接字符串是否正确
- 确认数据库服务器是否可访问
- 检查用户名和密码是否正确
- 确认数据库是否存在

### 2. 同步失败

**问题**：同步任务执行失败

**解决方案**：
- 查看日志文件了解详细错误信息
- 检查相关表是否存在（`vps_empinfo_mes`、`Sys_User`、`base_people`）
- 确认表结构是否正确
- 检查数据权限

### 3. 影响行数为0

**问题**：同步执行成功但影响行数为0

**说明**：这是正常情况，表示没有需要同步的数据

### 4. 程序无法启动

**问题**：运行 `dotnet run` 时出错

**解决方案**：
- 确认已安装 .NET 8.0 SDK
- 检查项目文件是否正确
- 确认所有依赖包已还原：`dotnet restore`

## 性能优化

1. **调整检查间隔**：根据实际需求调整 `CheckIntervalSeconds`
2. **禁用不需要的任务**：在配置中禁用不需要的同步任务
3. **数据库索引**：确保相关表有适当的索引
