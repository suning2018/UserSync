## 目录说明

### 根目录文件
- **Program.cs**: 应用程序入口点，包含主程序逻辑
- **UserSync.csproj**: .NET 项目文件，定义项目配置和依赖
- **UserSync.sln**: Visual Studio 解决方案文件
- **appsettings.json**: 应用程序配置文件，包含数据库连接、日志设置等

### Services/ - 服务层
包含所有业务逻辑服务类：
- **PersonnelSyncService**: 核心同步服务，负责执行所有同步任务
- **DatabaseService**: 数据库连接和基础操作服务
- **DatabaseLogService**: 数据库日志记录服务
- **DbHelper**: 数据库操作辅助类，封装常用数据库方法
- **UserPasswordService**: 用户密码管理服务
- **PasswordEncryptionHelper**: 密码加密工具类

### Models/ - 数据模型
包含应用程序使用的数据模型类：
- **AppSettings**: 强类型配置模型，包含所有配置选项

### Extensions/ - 扩展方法
包含各种扩展方法：
- **ServiceCollectionExtensions**: 依赖注入服务注册扩展
- **ConfigurationExtensions**: 配置读取和验证扩展

### Helpers/ - 辅助类
包含通用辅助工具类：
- **ConsoleHelper**: 控制台输出格式化辅助

### Constants/ - 常量定义
包含应用程序常量：
- **ConfigKeys**: 配置键名称常量

### SQL/ - SQL脚本
按功能分类的SQL脚本：
- **01_TableCreation**: 数据库表创建脚本
- **02_UserManagement**: 用户管理相关脚本
- **03_BasePeople**: base_people表管理脚本
- **04_DataSync**: 数据同步脚本
- **05_Diagnostic**: 诊断和比较脚本
- **06_Others**: 其他功能脚本

### Data/ - 数据文件
存放数据源文件：
- Excel文件、Markdown文件等数据源

### Docs/ - 文档
项目文档：
- 使用说明、技术文档、优化说明等

### Archive/ - 归档
已废弃或不再使用的代码文件

## 文件命名规范

### C# 文件
- 使用 PascalCase 命名
- 类名与文件名一致
- 一个文件一个类

### SQL 文件
- 使用小写字母和下划线
- 文件名应描述脚本功能
- 按功能分类到不同文件夹

### 文档文件
- 使用中文名称，便于理解
- Markdown 格式（.md）

## 最佳实践

1. **代码组织**：按功能模块组织代码，相关功能放在同一文件夹
2. **命名规范**：遵循 C# 命名规范，保持一致性
3. **文档维护**：及时更新文档，保持文档与代码同步
4. **版本控制**：使用 Git 管理代码，忽略 bin/ 和 obj/ 文件夹
5. **配置管理**：敏感信息不要提交到版本控制，使用配置文件模板
