# 用户登录信息生成说明

## 问题描述
通过SQL直接插入`Sys_User`表的数据，由于缺少`Sys_UserLogOn`表的对应记录，导致用户无法登录。

## 解决方案

### 方法1：使用SQL脚本生成基础记录（推荐）

执行 `SQL/02_UserManagement/generate_userlogon.sql` 脚本，为所有缺少登录信息的用户生成基础记录。

### 方法2：使用同步程序自动生成

运行人员同步程序，程序会自动执行 `GenerateUserLogOn` 任务，为缺少登录信息的用户生成 `Sys_UserLogOn` 记录。

## 密码加密说明

### 默认密码
- 默认密码：`123456`

### 加密步骤
密码需要经过以下三步加密：

1. **第一步：MD5加密（32位）**
   ```csharp
   string step1 = Md5.md5("123456", 32).ToLower();
   ```

2. **第二步：DES加密**
   ```csharp
   string encrypted = DESEncrypt.Encrypt(step1, F_UserSecretkey).ToLower();
   ```
   其中 `F_UserSecretkey` 是16位随机密钥（已通过SQL生成）

3. **第三步：再次MD5加密（32位）**
   ```csharp
   string finalPassword = Md5.md5(encrypted, 32).ToLower();
   ```
   这就是最终的 `F_UserPassword`

### 更新密码

密码更新需要在应用程序中完成，参考 `Archive/UpdateUserPasswords.cs` 中的实现。

## 相关文件

- `SQL/02_UserManagement/generate_userlogon.sql` - 生成登录信息SQL脚本
- `SQL/02_UserManagement/update_password_simple.sql` - 密码更新脚本（SQL方式，需CLR支持）
- `Archive/UpdateUserPasswords.cs` - C#密码更新代码示例

## 注意事项

1. **F_UserSecretkey**：已通过SQL自动生成16位随机密钥
2. **F_UserPassword**：需要在应用程序中通过加密逻辑生成后更新
3. **安全性**：建议首次登录后强制修改密码
