# 用户登录信息生成说明

## 问题描述
通过SQL直接插入`Sys_User`表的数据，由于缺少`Sys_UserLogOn`表的对应记录，导致用户无法登录。

## 解决方案

### 步骤1：生成Sys_UserLogOn基础记录

执行 `generate_userlogon.sql` 脚本，为所有缺少登录信息的用户生成基础记录。

```sql
-- 执行脚本后，会为每个用户生成：
-- - ID: 用户ID
-- - F_UserId: 用户ID
-- - F_UserSecretkey: 16位随机密钥（已生成）
-- - F_UserPassword: 空字符串（需要后续更新）
```

### 步骤2：生成加密密码（默认密码：123456）

由于密码加密逻辑复杂，需要在C#应用程序中完成。**默认密码为：123456**

以下是加密逻辑：

```csharp
// 默认密码：123456
string originalPassword = "123456";

// 第一步：对原始密码进行MD5加密（32位）
string step1 = JUWX.Utility.Md5.md5(originalPassword, 32).ToLower();

// 第二步：使用密钥进行DES加密
string encrypted = JUWX.Utility.DESEncrypt.Encrypt(step1, userSecretkey).ToLower();

// 第三步：对加密结果再次进行MD5加密（32位）
string finalPassword = JUWX.Utility.Md5.md5(encrypted, 32).ToLower();

// finalPassword 就是最终的 F_UserPassword
```

### 步骤3：批量更新密码为默认密码123456

已创建 `UpdateUserPasswords.cs` 文件，包含以下方法：

1. **`UpdateAllUserPasswordsToDefault()`** - 批量更新所有用户的密码为123456
2. **`UpdateUserPasswordToDefault(string userId)`** - 更新指定用户的密码为123456
3. **`GenerateUserLogOnWithDefaultPassword()`** - 生成登录信息并设置默认密码123456

**使用方法：**

```csharp
// 方法1：批量更新所有缺少密码的用户
UpdateAllUserPasswordsToDefault();

// 方法2：更新指定用户的密码
UpdateUserPasswordToDefault("用户ID");

// 方法3：为所有缺少登录信息的用户生成记录并设置默认密码
GenerateUserLogOnWithDefaultPassword();
```

**默认密码：123456**

### 步骤4：SQL批量更新（如果已有加密后的密码）

如果已经通过应用程序生成了加密后的密码，可以使用SQL批量更新：

```sql
-- 示例：批量更新密码（需要替换为实际的用户ID和加密后的密码）
UPDATE Sys_UserLogOn 
SET F_UserPassword = N'加密后的密码',
    F_LastModifyTime = GETDATE()
WHERE F_UserId = N'用户ID';
```

## 完整流程

### 方案A：分步执行（推荐）

1. **执行SQL脚本**：`generate_userlogon.sql`
   - 为所有缺少登录信息的用户生成基础记录
   - 自动生成16位随机密钥

2. **在应用程序中调用方法**：`UpdateAllUserPasswordsToDefault()`
   - 批量更新所有用户的密码为默认密码：**123456**

3. **验证**：
   - 检查所有用户是否都有登录信息
   - 测试用户登录功能（用户名：F_Account，密码：123456）

### 方案B：一步完成

直接在应用程序中调用：`GenerateUserLogOnWithDefaultPassword()`
- 自动为所有缺少登录信息的用户生成记录
- 自动设置默认密码为：**123456**

## 注意事项

1. **默认密码**：所有新用户的默认密码为 **123456**
2. **密钥生成**：每个用户的`F_UserSecretkey`是唯一的，不能重复使用
3. **密码加密**：必须严格按照提供的三步加密逻辑，否则无法登录
4. **安全性**：
   - 建议首次登录后强制修改密码
   - 生产环境建议使用更复杂的默认密码策略
   - 可以通过修改`UpdateUserPasswords.cs`中的`defaultPassword`变量来更改默认密码

## 查询缺少登录信息的用户

```sql
SELECT 
    u.ID,
    u.F_Account,
    u.F_RealName,
    u.F_EnabledMark
FROM Sys_User u
LEFT JOIN Sys_UserLogOn ul ON u.ID = ul.F_UserId
WHERE ul.ID IS NULL
    AND u.F_DeleteMark = 0
    AND u.F_EnabledMark = 1;
```

## 验证登录信息完整性

```sql
SELECT 
    COUNT(*) AS TotalUsers,
    COUNT(ul.ID) AS UsersWithLogon,
    COUNT(*) - COUNT(ul.ID) AS UsersWithoutLogon
FROM Sys_User u
LEFT JOIN Sys_UserLogOn ul ON u.ID = ul.F_UserId
WHERE u.F_DeleteMark = 0
    AND u.F_EnabledMark = 1;
```
